# 🏢 기업성장 브릿지 Map 운영자 매뉴얼

이 매뉴얼은 '기업성장 브릿지 Map' 서비스의 데이터를 관리하고, 최신 정보를 배포하는 운영자를 위해 작성되었습니다.

---

## 📑 1. 구글 시트 데이터 관리 (Data Management)

서비스의 모든 기업 정보는 지정된 [구글 시트](https://docs.google.com/spreadsheets/d/1ho-RJbCDEeWkfp1XgFm0M2QGyVNBnH6KbzXUR_nwMts/edit?usp=sharing)를 통해 관리됩니다.

### ✅ 데이터 입력 규칙
1.  **ID (고유 식별자):** 영어 소문자와 숫자, 하이픈(`-`)만 사용하세요. (예: `samsung-electronics`, `yyg-startup-01`)
    *   *주의: 한글이나 공백은 절대 사용하지 마세요.*
2.  **도로명 주소:** 정확한 도로명 주소를 입력해야 지도의 마커 위치가 정확하게 잡힙니다.
3.  **게시 상태 (`map_display_status`):**
    *   `VISIBLE`: 서비스에 즉시 노출
    *   `HIDDEN`: 데이터는 유지하되 지도에서 숨김
    *   `DRAFT`: 작성 중 (노출 안 됨)
4.  **다중 항목 입력:** `certifications`(인증항목) 등 여러 값을 입력할 때는 쉼표(`,`)로 구분하세요. (예: `지역우수,청년도약`)

---

## 🖼 2. 이미지 에셋 등록 (Logos & Images)

이미지 파일은 프로젝트의 특정 폴더에 업로드해야 합니다.

### 🏢 기업 로고 (`public/assets/logos/`)
*   **파일명 규칙:** 구글 시트의 `logo` 컬럼에 입력한 파일명과 **토씨 하나 틀리지 않게** 동일해야 합니다.
*   **권장 사양:** 400x400px 이상, 정사각형 비율, 배경이 투명한 PNG 또는 흰색 배경의 JPG/PNG.

### 📸 기업 홍보 이미지 (`public/assets/companies/`)
*   **파일명 규칙:** 구글 시트의 `images` 컬럼에 입력한 파일명들입니다. (여러 장일 경우 쉼표로 구분)
*   **권장 사양:** 16:9 비율 (예: 1280x720px), JPG 또는 WebP 포맷 권장.

---

## 🔄 3. 데이터 동기화 및 업데이트 (Sync)

구글 시트의 내용을 서비스 데이터(`companies.json`)로 변환하는 과정입니다.

### 방법 A: GitHub Actions 활용 (권장 - 무설치)
1.  GitHub 저장소의 **[Actions]** 탭으로 이동합니다.
2.  왼쪽 메뉴에서 **"Sync Google Sheets Data"** 워크플로우를 선택합니다.
3.  **[Run workflow]** 버튼을 클릭하여 실행합니다.
4.  완료되면 데이터가 자동으로 업데이트되고 서비스에 반영(배포)됩니다.

### 방법 B: 로컬 환경에서 실행 (개발자용)
```bash
# 시트 데이터 가져오기
npm run sync

# 변경사항 확인 후 커밋 & 푸시
git add .
git commit -m "data: 기업 정보 업데이트"
git push origin main
```

## 🚀 4. 배포 (Deployment)

본 서비스는 **GitHub Pages**를 통해 호스팅됩니다.

*   `main` 브랜치에 코드가 푸시되거나, 위의 **Sync 워크플로우**가 성공적으로 끝나면 자동으로 배포 프로세스가 시작됩니다.
*   약 2~3분 후 [서비스 URL]에서 변경된 내용을 확인할 수 있습니다.

## ⚠️ 운영 시 유의사항 (Critical Notes)

1.  **주소 오타 주의:** 주소가 잘못되면 지도가 해당 기업을 찾지 못해 엉뚱한 곳(또는 바다 한가운데)에 표시될 수 있습니다.
2.  **대소문자 구분:** 파일명(logo, images)은 대소문자를 엄격히 구분합니다. `Logo.png`와 `logo.png`는 서로 다른 파일로 인식됩니다.
3.  **시트 구조 유지:** 구글 시트의 첫 번째 행(헤더)은 프로그램이 데이터를 읽는 기준입니다. 열의 순서를 바꾸거나 이름을 임의로 수정하지 마세요.
4.  **데이터 백업:** 대량의 데이터를 수정하기 전에는 반드시 기존 시트를 복사하여 백업해 두는 습관을 가지세요.
